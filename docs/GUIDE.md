## 快速开始

### 前提条件

- Windows 10/11 (x64)
- AOC 显示器（USB 上游线已连接至电脑）
- 从 USB 口供电且与电脑建立通讯

### 运行 CLI

```bash
# 查看帮助
aoc --help

# 查询所有信息
aoc --info

# 设置响应速度为 "strong"
aoc --set overdrive strong

# 读取当前 HDR 模式
aoc --get hdr
```

### 运行 GUI

直接运行 `AOC.UI.exe` 启动图形界面。程序会自动启动后台代理进程并连接显示器。

## CLI 用户手册

### 全局用法

```
AOCMenu --set <设置名> <值>
AOCMenu --get <设置名>
AOCMenu --help
AOCMenu --info [子命令]
```

| 参数 | 说明 |
|---|---|
| `--set <key> <value>` | 写入设置值。`key` 为设置键名（小写），`value` 为期望值 |
| `--get <key>` | 读取当前设置值。返回值的显示名称（如有映射）和原始 SDK 值 |
| `--help` / `-h` / `/help` / `/h` | 显示帮助信息及所有可用设置 |
| `--info [topic]` | 查询显示器状态信息。不带 topic 时查询全部；带 topic 查询单项 |

### 可用设置

#### 图像

| 设置键 | 中文名 | SDK 方法 | 可选值 |
|---|---|---|---|
| `gamma` | Gamma 值 | `SetGamma` | `1` `2` `3` |
| `low-blue` | 低蓝光模式 | `SetLowBlueModel` | `关闭` `多媒体` `网络` `办公室` `阅读` |
| `overdrive` | 响应速度 | `SetOverDrive` | `off` `weak` `medium` `strong` `strongest` |

#### 色彩

| 设置键 | 中文名 | SDK 方法 | 可选值 |
|---|---|---|---|
| `color-gamut` | 色彩空间 | `SetColorGamut` | `standard` `srgb` |

#### HDR

| 设置键 | 中文名 | SDK 方法 | 可选值 |
|---|---|---|---|
| `hdr` | HDR 模式 | `SetHDR` | `关闭` `DisplayHDR` `图片` `电影` `游戏` |

### 设置依赖关系

部分设置之间存在硬件层面的依赖关系，当依赖条件不满足时操作会被拒绝并提示原因。

| 设置 | 依赖条件 | 说明 |
|---|---|---|
| `gamma` | 需要 `color-gamut` 为 `standard` | 切换到 srgb 色彩空间时，Gamma 和低蓝光会被硬件强制固定 |
| `low-blue` | 需要 `color-gamut` 为 `standard` | 同上 |
| `color-gamut` | 需要 `hdr` 为 `关闭` | 开启 HDR 后色彩空间不可调整 |
| `hdr` | 需要 `color-gamut` 为 `standard` | 切换到 srgb 后 HDR 模式不可调整 |

**典型场景**: 若需调节 Gamma，请先确保 `color-gamut` 为 `standard`（即 srgb 关闭）、且 `hdr` 为 `关闭`。

### 信息查询

通过 `--info [topic]` 查询显示器只读状态信息：

| 子命令 | SDK 方法 | 显示内容 | 推荐 |
|---|---|---|---|
| `message` | `GetDisPlayMessage` | 显示器型号、制造商、固件版本、EDID 信息 | ⭐ |
| `list` | `GetDisPlayList` | 已连接显示器列表 | |
| `name` | `GetCurDisplayName` | 当前选中的显示器名称 | |
| `device` | `GetConnectDevice` | 已连接的设备信息 | |
| `support` | `GetSupport` | SDK 支持的功能列表 | |
| `oled` | `GetOLEDPanelCareInfo` | OLED 像素刷新/位移保养信息 | |

> `--info`（无 topic）查询所有项目，适合一次性获取显示器完整状态。

#### 输出示例

```
ℹ️ 显示器型号/固件版本
──────────────────────────
  err_code: 0
  IsSucc: true
  err_msg: (null)
  Tag:
    sManufacturer: AOC
    sMonitorName: CU34G3X
    sSerialNumber: ABC123
    sVersion: V1.0
    ScreenSize: ~34.1"
    TimingRecommandation: 3440x1440
```

### 退出码

| 退出码 | 含义 |
|---|---|
| `0` | 操作成功 |
| `1` | 运行时错误（初始化失败、SDK 调用失败） |
| `2` | 用户输入错误（未知设置、非法值、依赖条件不满足） |

### 完整示例

```bash
# ── 查看帮助 ──
aoc --help

# ── 读取设置 ──
aoc --get overdrive
# 输出: 响应速度 OverDrive: strong (2)

aoc --get hdr
# 输出: HDR 模式: 关闭 (0)

# ── 写入设置 ──
aoc --set overdrive strong
# 输出: 响应速度 OverDrive → strong... ✅

aoc --set hdr 游戏
# 输出: HDR 模式 → 游戏... ✅

# ── 信息查询 ──
aoc --info
# 查询所有显示器信息

aoc --info message
# 仅查询显示器型号/固件

aoc --info oled
# 仅查询 OLED 保养信息
```

## GUI 用户手册

### 主界面

运行 `AOC.UI.exe` 启动图形界面。界面分为以下几部分：

#### 状态栏
页面顶部显示：
- **进度环** — 代理连接和初始化期间旋转
- **状态文本** — 显示当前连接状态（"正在连接代理..." / "已连接" / 错误信息）
- **刷新按钮** (🔄) — 重新读取所有设置的当前值
- **设置按钮** (⚙️) — 打开应用设置页

#### 设置分组
所有设置按类别分组显示：
- **图像** — Gamma、低蓝光模式、响应速度 OverDrive
- **色彩** — 色彩空间
- **HDR** — HDR 模式

每个设置项为一个卡片（SettingsCard）：
- **枚举类型**（如 HDR 模式）→ 下拉选择框（ComboBox）
- **数值范围**（如数值类设置）→ 滑块（Slider）+ 数值显示
- **只读** → 纯文本显示当前值

#### 依赖关系自动处理
GUI 会自动检测设置间的依赖关系并反馈：
- 当一个设置因依赖条件不满足而禁用时，卡片会变灰并显示提示（如 "⚠️ 需要 色彩空间 为 standard"）
- 修改依赖设置（如 `color-gamut`）后，受影响设置自动重新加载

### 应用设置页

点击主界面右上角的齿轮按钮 (⚙️) 打开应用设置页。

#### 连接状态
- **代理连接** — 显示与 ZeasnProxy 后台服务的连接状态（绿点=已连接，灰点=未连接）
- **管道名称** — 命名管道通讯标识（ZeasnProxy）

#### 显示器信息
- **概要卡片** — 显示制造商、型号、尺寸和推荐分辨率
- **详细信息** — 等宽字体显示的完整 EDID 信息，包括序列号、固件版本、Gamma、色度坐标等
- **刷新按钮** — 重新读取显示器信息

#### 主题切换
- **跟随系统** — 自动匹配 Windows 主题
- **浅色** — 固定浅色主题
- **深色** — 固定深色主题

切换主题时伴随**圆形扩散动画**：颜色从下拉菜单中心向外扩散覆盖整个窗口，视觉效果流畅。

#### 关于
显示应用名称和版本号。

### 系统托盘

- **关闭窗口** → 最小化到系统托盘，程序继续后台运行
- **系统托盘图标**：🖥️ 或自定义 AppIcon
- **右键菜单**：
  - **显示主窗口** — 恢复窗口显示
  - **退出** — 发送关闭指令给 ZeasnProxy 后完全退出

### 窗口置顶

主窗口标题栏右侧（关闭按钮左侧）有**图钉按钮** 📌：
- 点击切换窗口置顶/取消置顶
- 置顶后窗口始终显示在其他窗口之上，适合游戏或全屏应用场景

## 常见问题

### Q: 运行后提示 "显示器初始化失败"

**可能原因**:
1. 显示器 USB 上游线未连接（需使用 USB-B 上行线连接电脑）
2. 显示器型号不兼容（仅支持 AOC 部分型号的 USB 私有协议）
3. ZeasnProxy 加载 SDK 失败（可尝试重启）
4. 权限不足（确保以当前用户运行）

**排查步骤**:
```bash
# 1. 查看帮助验证程序正常
aoc --help

# 2. 查询显示器信息（如失败会给出诊断信息）
aoc --info message

# 3. 如果上述失败，检查 ZeasnProxy 是否存活
# 打开任务管理器查看 ZeasnProxy.exe 进程
```

### Q: `--set` 报错 "当前状态无法调整"

说明该设置当前受依赖设置限制。请按提示调整相关设置后再试。详细依赖关系见上方[设置依赖关系](#设置依赖关系)表格。

### Q: 如何同时调节多个参数？

CLI 设计为单次操作一个参数。如需批量设置，可编写脚本：

```bash
aoc --set color-gamut standard
aoc --set gamma 2
aoc --set overdrive medium
```

### Q: GUI 和 CLI 可以同时使用吗？

可以。它们共享同一个 ZeasnProxy 代理进程，互不干扰。GUI 修改设置后，CLI 查询会反映最新值。

### Q: 修改设置后 OSD 菜单没有立即反映变化？

显示器 OSD 可能需要短暂刷新。物理按下显示器按键进入 OSD 菜单即可查看最新值。部分设置需要显示器重启或重新切换信号源后生效（由显示器固件决定）。

### Q: GUI 窗口不见了？

点击系统托盘图标（🖥️）重新显示。如果托盘图标也消失，可能是程序已退出。可重新运行 `AOC.UI.exe`。

### Q: 设置值在显示器 OSD 中和软件中不一致？

SDK 查到的值反映的是显示器芯片内部的寄存器状态，和 OSD 菜单可能有不同的默认值/取值范围。如遇明显不一致，可以：

1. `aoc --get <key>` 读取软件端值
2. 对比显示器 OSD 显示
3. 如确认为 Bug，请反馈
